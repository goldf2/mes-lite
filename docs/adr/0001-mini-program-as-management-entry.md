# ADR 0001: 小程序第一版作为移动管理入口

## 状态

已接受

## 背景

MiniERP 需要接入微信小程序。小程序可以放在开发商主体下，也可能后续迁移到客户公司主体。

如果第一版同时承载内部管理、消费者商城、微信支付和客户公司对外经营行为，会增加微信审核、主体迁移、数据归属和支付结算风险。

## 决策

第一版小程序定位为企业内部移动管理入口。

包含：

- 查库存
- 采购入库
- 销售出库
- 库存盘点
- 单据查询
- 经营摘要

不包含：

- 消费者商城
- 在线支付
- 会员营销
- 客户公司对外履约服务

第一版允许注册在开发商主体下，但业务数据必须归属于租户。系统不得通过小程序 AppID 判断业务数据归属。

## 影响

正面影响：

- 降低微信审核和类目资质复杂度。
- 后续更容易进行小程序主体变更或客户独立部署。
- 数据模型可以先围绕内部管理闭环稳定下来。

负面影响：

- 第一版不能直接支持客户面向消费者经营。
- 如果客户需要独立小程序，需要额外做 AppID 配置和用户重新绑定流程。

## 后续条件

当 MiniERP 需要支持 C 端销售、微信支付、会员或客户公司品牌前台时，应新增独立 ADR，重新判断是否使用客户公司主体小程序。

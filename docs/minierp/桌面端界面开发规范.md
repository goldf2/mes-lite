# MES-lite 桌面端界面开发规范

桌面端面向电脑大屏、鼠标键盘、长时间录入和批量处理。核心目标是信息密度高、操作入口清楚、对比方便、重复操作效率高。

## 1. 适用范围

- 默认适用于宽度大于等于 1024px 的电脑浏览器。
- 1280px 到 1920px 是主要设计区间。
- 桌面端不按手机逻辑隐藏大量功能，不能为了移动端简化而牺牲大屏效率。
- 具体断点和截图验收要求见：`docs/minierp/响应式断点与验收矩阵.md`。

## 2. 页面结构

- 左侧固定业务导航，显示主要业务模块。
- 顶部固定工具栏，承载关键词、筛选、视图切换、主操作、用户菜单。
- 页面主体优先使用表格、左右分栏、详情面板、统计卡片。
- 表格和详情可以并列展示时，优先并列，不要强制进入全屏弹窗。
- 仪表盘和统计分析使用图表、指标卡、预警列表，不使用普通业务列表模板。

## 3. 顶部工具栏

- 桌面端首层必须尽量暴露高频功能。
- 关键词搜索放左侧或标题后方。
- 筛选按钮和筛选摘要放关键词之后。
- 新增、导入、导出、调整、打印等主操作放右侧。
- 视图切换按钮放主操作左侧。
- 用户菜单固定在最右侧。
- 顶部工具栏可以换行，但不能出现页面级横向滚动条。
- 高频按钮不进入“更多”菜单，除非一行无法承载并且该按钮低频。

## 4. 筛选模式

- 桌面端允许首层显示关键词、核心客户筛选、核心状态筛选。
- 复杂筛选统一进入筛选弹窗。
- 筛选弹窗可以是浮层，宽度建议 480px 到 640px。
- 筛选弹窗内允许多列布局，减少垂直滚动。
- 筛选结果必须显示摘要，例如“筛选 3”或标签摘要。
- 筛选项不能在页面上拆成多排杂乱按钮。

## 5. 表格优先原则

- 桌面端业务列表优先使用表格。
- 表格列顺序一般为：预览、编码/单号、名称/标题、类型、状态、关联对象、数量/金额、凭证/附件、操作。
- 操作列固定最右侧。
- 行内操作顺序固定：全屏打开 / 详情 / 编辑 / 上传 / 打印 / 归档。
- 表格行高保持紧凑但可读，普通行 48px 到 64px，附件预览行可到 72px。
- 表格允许自身横向滚动，但不得撑破页面。
- 大字段可以截断并提供详情，不要把表格列拉到不可读。

## 6. 卡片使用

- 桌面端卡片只用于仪表盘、统计指标、实体摘要、图片资料。
- 桌面端业务列表可以提供卡片视图，但不是默认必需。
- 卡片不能嵌套卡片。
- 卡片内信息顺序固定：编码/状态、名称、关键属性、数量/金额、操作。
- 卡片操作放底部右侧。

## 7. 详情与编辑

- 桌面端详情优先使用大弹窗或右侧详情面板。
- 详情面板宽度建议 720px 到 1200px。
- 编辑状态在详情内切换，不再打开第二层弹窗。
- 表单字段按 2 到 3 列布局。
- 长备注、附件、操作记录独立成区块。
- 新增表单不能是一长条，必须用 `FormGrid`。
- 保存、取消按钮固定在弹窗底部右侧。

## 8. 文件与图片

- 桌面端文件查看优先使用全屏查看器。
- 图片和 PDF 统一使用 `FileViewer`。
- 列表首层需要直接提供“全屏打开”。
- 文件详情内仍可提供“打开”，但打开的是同一个查看器。
- 图片查看按钮顺序固定：上一份、下一份、缩小、放大、左转、右转、复位、新窗口、关闭。

## 9. 按钮尺寸

- 顶部工具栏按钮高度 36px 到 40px。
- 表格行内按钮高度 28px 到 32px。
- 弹窗底部按钮高度 36px 到 40px。
- 同一行按钮必须同高、同圆角、同字号。
- 文字按钮优先用于明确命令，图标按钮必须有 tooltip 或可理解图标。

## 10. 桌面端禁止事项

- 禁止把主要业务按钮全部收到二级菜单。
- 禁止用手机卡片布局替代所有桌面表格。
- 禁止表格列无序堆叠。
- 禁止一行按钮高度、颜色、文案不统一。
- 禁止在桌面端让用户频繁进入全屏弹窗才能完成基础比对。
- 禁止页面主体出现横向滚动条。

## 11. 桌面端验收清单

1. 1280px 宽度下顶部工具栏是否清楚。
2. 主要操作是否暴露在首层。
3. 表格是否能完成扫描、对比、批量判断。
4. 行内操作是否在最右列且顺序统一。
5. 详情和编辑是否在同一面板内完成。
6. 附件查看是否使用统一 `FileViewer`。
7. 页面主体是否没有横向滚动条。
8. 长时间操作时是否减少不必要弹窗层级。
9. 1024、1366、1440、1920 宽度下是否均完成实际页面模拟。

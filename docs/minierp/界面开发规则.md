# MES-lite 界面开发规则

本文用于后续界面开发前自查，避免重复破坏已经确定的交互规则。

## 1. 顶部工具栏

- 常用筛选必须放在首层直接显示，不能默认藏进“筛选”二级菜单。
- 关键词搜索、状态勾选、类别勾选属于常用筛选，应优先放在首层。
- 客户、供应商、物料、文件类型、排序、字段显示等可以作为高级筛选；空间不足时才折叠到“更多筛选”。
- 新增、导入、导出、上传、调整等高频操作按钮必须暴露在首层。
- 顶部工具栏不得产生页面级横向滚动条；窄屏时优先换行、压缩宽度或折叠高级筛选。

## 2. 筛选控件

- 勾选类筛选使用可见复选框，不用只显示一个汇总按钮。
- 勾选项数量较多时允许在工具栏内换行显示。
- 筛选状态应尽量保存在浏览器本地，避免刷新后丢失用户习惯。
- 物料、客户、供应商等数据量可能较大的选项，优先使用“输入搜索后选择”，避免长下拉框。
- 搜索选择不能只依赖前端已加载的少量数据；输入关键字时应调用后端查询。

## 3. 模块入口与操作入口

- 业务模块入口放在左侧业务导航；系统管理类入口放在顶部或系统菜单。
- 查看、编辑、上传、归档等常用行操作应直接暴露，不能只藏在二级菜单。
- 展示型模块不应把查看、上传、编辑拆成多个并列入口；列表只保留一个“打开/展示”入口，在同一展示面板内完成上传和基础信息维护。
- 展示型模块需要大屏查看时，优先提供“全屏显示”开关，而不是在列表层堆多个查看按钮。
- 上传类功能必须在详情页有明确的大号入口；如果新建后才能上传，应在保存后自动进入上传区域。
- “删除”语义默认改为归档或软删除，界面文案避免直接写不可恢复的删除。

## 4. 移动端与窄屏

- 页面主容器不能出现横向滚动条。
- 表格可以在自身容器内横向滚动，但不能撑破整个页面。
- 手机端优先卡片视图，列表视图应保持为辅助选择。
- 工具栏在窄屏可换行；高级筛选可折叠，但关键词和核心状态筛选仍应尽量可见。

## 5. 新模块开发前检查

1. 是否有首层关键词筛选。
2. 是否有首层状态或类别勾选筛选。
3. 是否把高频操作按钮放在首层。
4. 是否避免页面级横向滚动。
5. 是否支持卡片 / 列表视图或明确说明为什么不需要。
6. 是否支持软删除 / 归档与恢复路径。
7. 是否完成 `npx tsc --noEmit --pretty false` 和 `npm run build` 验证。

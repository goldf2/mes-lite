# MES-lite 界面草图到实现流程

本文定义界面从草图到实际工程落地的协作流程。目标是先明确页面结构和操作优先级，再写代码，再用多分辨率模拟验证，而不是边写页面边临时调整。

## 1. 推荐流程

```txt
用户草图
  -> 草图解释与页面规范化
  -> 低成本工程实现
  -> 多分辨率模拟测试
  -> 根据测试结果修正布局
  -> 抽取公共组件
  -> 提交、推送、部署验证
```

这个流程是正确的，尤其适合 MES/ERP 这种多模块、多表格、多操作按钮的系统。

## 2. 用户草图阶段

用户先给出草图，可以是：

- 手绘照片。
- 截图标注。
- Figma / Axure / Excalidraw。
- 简单文字框架。
- 现有页面截图 + 需要修改的说明。

草图不需要精确到像素，但需要表达以下信息：

1. 这个页面解决什么业务问题。
2. 桌面端最重要的 3 到 5 个操作是什么。
3. 手机端最重要的 1 到 2 个操作是什么。
4. 哪些字段必须首屏显示。
5. 哪些字段可以进入详情。
6. 哪些筛选条件必须常用。
7. 哪些操作需要弹窗、全屏、底部菜单。
8. 是否有附件、图片、PDF、原始凭证。
9. 是否有审核、归档、恢复、权限限制。

## 3. 草图解释阶段

Codex 根据草图输出一份页面落实方案：

- 页面属于哪种模板：列表页、详情页、单据页、全景页、统计页、设置页。
- 桌面端布局如何：表格优先、分栏、详情弹窗或右侧面板。
- 移动端布局如何：卡片优先、操作菜单、底部抽屉、全屏详情。
- 顶部工具栏放什么。
- 行内操作放什么。
- 哪些功能进入“更多”。
- 哪些字段进入列表，哪些字段进入详情。
- 需要复用哪些公共组件。
- 需要新增哪些公共组件。

这个阶段如果发现草图不适合某个分辨率，应先说明问题，再给出调整版布局。

## 4. 工程实现阶段

实现时遵循以下顺序：

1. 先用现有公共组件实现页面骨架。
2. 不急着追求一次性完美，先完成可运行垂直切片。
3. 页面按钮、表格、卡片、弹窗不能自由写样式，应尽量复用统一组件。
4. 如果发现重复样式超过两处，应优先抽成公共组件。
5. 业务数据和视觉组件分离，页面只传入数据、状态和操作函数。

## 5. 多分辨率模拟测试阶段

实现后必须按 `docs/minierp/响应式断点与验收矩阵.md` 做实际模拟。

至少检查：

- 360 x 800
- 390 x 844
- 414 x 896
- 768 x 1024
- 1024 x 768
- 1366 x 768
- 1440 x 900
- 1920 x 1080

每个尺寸都要看：

- 页面是否横向滚动。
- 顶部按钮是否重叠。
- 筛选是否可用。
- 操作菜单是否能打开和关闭。
- 卡片或表格是否可读。
- 详情弹窗是否遮挡或超出。
- 文件查看器是否可用。

## 6. 修正阶段

模拟测试发现问题后，修正优先级如下：

1. 页面级横向滚动。
2. 按钮重叠或不可点击。
3. 核心字段不可读。
4. 主要操作不可发现。
5. 弹窗无法关闭或内容溢出。
6. 手机端卡片过高或按钮过多。
7. 桌面端信息密度不足。

修正应尽量发生在公共组件或页面模板里，而不是每个页面分别打补丁。

## 7. 组件沉淀阶段

当同类结构出现两次以上，必须考虑沉淀公共组件：

- 同类按钮 -> `Button`
- 同类状态 -> `StatusBadge`
- 同类筛选 -> `FilterPopover`
- 同类列表 -> `DataTable`
- 同类卡片 -> `EntityCard`
- 同类详情 -> `DetailPanel`
- 同类文件查看 -> `FileViewer`

公共组件沉淀后，旧页面应逐步迁移，避免新旧样式长期并存。

## 8. 交付阶段

每次界面开发完成后，至少交付：

- 实现内容说明。
- 验证过的分辨率。
- 仍存在的问题。
- 是否修改了公共组件。
- 是否影响其他页面。
- `npx tsc --noEmit --pretty false` 结果。
- `npm run build` 结果。

如果是 bug 修复，用户确认有效后，需要按项目规则写入修复报告。

## 9. 草图提交模板

用户给草图时，可以按以下模板补充说明：

```txt
页面名称：
主要使用场景：
桌面端最重要操作：
手机端最重要操作：
必须首屏显示字段：
可进入详情字段：
筛选条件：
行内操作：
附件/图片/PDF：
权限限制：
参考截图或草图：
```

## 10. 判断标准

一个草图不是最终界面，它只是表达业务优先级。最终落地必须同时满足：

- 符合业务流程。
- 符合桌面端规范。
- 符合移动端规范。
- 符合响应式断点验收。
- 可以沉淀为可复用组件。
- 后续修改可以通过公共组件同步影响多个页面。

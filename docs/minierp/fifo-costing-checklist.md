# FIFO 成本核算验收记录

日期：2026-07-12

## 已实现范围

- 来料确认收货后生成 `InventoryCostLayer` 成本层。
- FIFO 物料领料时按成本层创建 `CostLayerConsumption` 消耗明细。
- 跨批次领料时先消耗最早成本层，再消耗后一成本层。
- 领料同时扣减库存数量、核算数量、库存金额、平均单位成本。
- 工单取消时按消耗明细恢复 FIFO 成本层，并把已领物料退回库存。
- 工单创建时记录 `reservedValuationQty`，避免后续库存平均比率变化导致释放预留不准。
- FIFO 物料暂不允许直接盘点调整，防止库存数量与成本层脱节。

## 自动化测试

执行命令：

```bash
npx tsx scripts/verify-fifo.ts
```

测试场景：

1. 创建 FIFO 原料，两批入库：
   - 第一批：10 根，20 kg，总成本 100。
   - 第二批：10 根，30 kg，总成本 180。
2. 创建工单预留 12 根、30 kg。
3. 领料 12 根：
   - 第一层消耗 10 根、20 kg、成本 100。
   - 第二层消耗 2 根、6 kg、成本 36。
   - 合计消耗 12 根、26 kg、成本 136。
4. 检查领料后库存：
   - 剩余 8 根、24 kg、库存金额 144。
   - 第一层关闭，第二层剩余 8 根、24 kg、金额 144。
5. 模拟取消退料：
   - 库存恢复为 20 根、50 kg、金额 280。
   - 两个成本层均恢复为 OPEN。
   - 消耗明细标记为已恢复。

通过标准：

- 脚本输出 `FIFO 验证通过`。
- `npx prisma validate` 通过。
- `npx prisma migrate status` 显示数据库结构最新。
- `npm run build` 通过。

## 当前限制

- FIFO 已覆盖来料入库、工单领料、工单取消退料主链路。
- FIFO 物料的盘点调整被暂时禁止，需要后续新增“期初成本层 / 盘盈 / 盘亏”单据来处理。
- 生产报废、材料退料、成品成本归集还需要后续独立建模。
- `npm run lint` 当前会触发 Next.js ESLint 初始化交互，项目尚未配置非交互 lint。

# MiniERP 领域模型

## 建模原则

1. 先做内部管理后台，不做消费者商城。
2. 以库存流水为核心事实，当前库存由流水汇总或快照维护。
3. 所有业务数据必须带租户边界。
4. 小程序账号信息只作为接入配置，不作为业务数据归属依据。
5. 第一版宁可少功能，也要把采购、销售、库存三个闭环跑通。

## 模块边界

### 1. 租户与组织

用于描述谁在使用系统。

核心对象：

- 租户
- 员工
- 角色
- 权限
- 门店
- 仓库

第一版建议角色：

- 超级管理员：开发商运维账号，不能默认查看客户业务数据。
- 租户管理员：客户公司负责人。
- 提交人员：录入并提交业务内容。
- 审核人员：审核注册人员和业务提交内容。
- 管理人员：管理人员、权限和系统配置。
- 仓管：入库、出库、盘点、查库存。
- 销售：客户、报价、销售单。
- 采购：供应商、采购单、入库。
- 财务查看：只读报表，不做会计系统。

操作人员注册流程：

1. 操作人员提交注册。
2. 首位注册人员自动成为管理人员，用于初始化系统。
3. 后续注册人员进入待审核状态。
4. 审核人员或管理人员审核通过后，账号才能登录系统。
5. 管理人员可以调整角色或停用账号。

### 2. 商品与库存基础资料

用于描述系统管理什么货。

核心对象：

- 商品分类
- 品牌
- 商品
- SKU
- 单位
- 条码
- 价格
- 库存预警规则

商品和 SKU 的关系：

- 商品是业务名称，例如“白色 T 恤”。
- SKU 是库存单位，例如“白色 T 恤 / M 码 / 纯棉”。
- 第一版可以让一个商品默认对应一个 SKU，保留多规格扩展位。

### 3. 往来单位

用于描述和谁交易。

核心对象：

- 客户
- 供应商
- 联系人
- 地址

第一版客户和供应商可以共用一张往来单位表，通过类型区分。

### 4. 采购

用于描述从供应商进货。

核心对象：

- 采购订单
- 采购订单明细
- 采购入库单
- 采购入库明细

第一版流程：

1. 创建采购订单。
2. 采购订单审核。
3. 到货后创建采购入库单。
4. 入库确认后生成库存流水。
5. 采购订单根据入库数量更新状态。

采购订单状态：

- 草稿
- 待审核
- 已审核
- 部分入库
- 已完成
- 已取消

### 5. 销售

用于描述向客户出货。

核心对象：

- 销售订单
- 销售订单明细
- 销售出库单
- 销售出库明细

第一版流程：

1. 创建销售订单。
2. 检查可用库存。
3. 销售订单审核。
4. 创建销售出库单。
5. 出库确认后生成库存流水。
6. 销售订单根据出库数量更新状态。

销售订单状态：

- 草稿
- 待审核
- 已审核
- 部分出库
- 已完成
- 已取消

### 6. 库存

用于描述货在什么地方、数量如何变化。

核心对象：

- 库存余额
- 库存流水
- 库存调整单
- 盘点单
- 调拨单

库存流水类型：

- 采购入库
- 销售出库
- 盘盈入库
- 盘亏出库
- 手工调整入库
- 手工调整出库
- 调拨出库
- 调拨入库

库存数量建议分层：

- 现存量：仓库里实际有多少。
- 锁定量：已被销售订单占用但未出库的数量。
- 可用量：现存量减锁定量。

第一版可以先不做锁定量，但数据模型保留字段。

### 7. 报表

第一版只做经营摘要，不做财务总账。

基础报表：

- 当前库存表
- 库存流水表
- 低库存预警
- 采购汇总
- 销售汇总
- 商品销售排行

### 8. 原始单据附件

用于保存业务动作背后的原始凭证，通常是单据照片，也可以是 PDF。

核心对象：

- 附件
- 业务对象类型
- 业务对象 ID
- 上传人

附件适用对象：

- 采购入库单
- 销售出库单
- 盘点单
- 调整单
- 调拨单
- 生产工单
- 来料单
- 发货单
- 退货单

附件不改变业务状态，只用于追溯和审计。删除附件不应删除业务单据。

### 9. 微信小程序接入

小程序作为移动端入口，不作为业务主体。

核心对象：

- 小程序应用
- 微信用户绑定
- 员工登录会话
- 租户小程序配置
- 原始单据照片上传

登录关系：

1. 微信用户通过小程序登录。
2. 系统拿到 openid 或 unionid。
3. 用户绑定到某个租户员工。
4. 员工按角色获得权限。

一个微信用户可以绑定多个租户员工身份，但默认进入最近使用的租户。

## 核心业务闭环

### 查库存

员工选择租户和仓库，按商品名、SKU、条码搜索库存余额。系统展示现存量、锁定量、可用量和最近库存流水。

### 采购入库

采购或仓管创建入库单，选择供应商、仓库和 SKU，确认后写入采购入库流水，并增加库存余额。

### 销售出库

销售或仓管创建出库单，选择客户、仓库和 SKU，确认后写入销售出库流水，并减少库存余额。

### 库存盘点

仓管创建盘点单，录入实盘数量。系统计算账面数量和差异，审核后按差异生成盘盈或盘亏流水。

## 多租户边界

所有业务表必须包含 `tenant_id`。

禁止通过小程序 AppID 判断数据归属。AppID 只是入口，用户进入后必须落到租户身份。

开发商运维账号需要单独授权和审计，不能默认跨租户查询客户业务数据。

## 后续扩展点

- 客户独立小程序主体。
- 微信支付和客户自己的商户号。
- 多门店零售前台。
- 简易生产领料和完工入库。
- 扫码出入库。
- Excel 导入导出。
- 数据备份和客户迁出包。
